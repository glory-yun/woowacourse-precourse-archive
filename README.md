# javascript-lotto-precourse

# 💰로또

# 기능 요구사항

- 로또 번호의 숫자 범위는 1~45까지이다.
- 1개의 로또를 발행할 때 중복되지 않는 6개의 숫자를 뽑는다.
- 당첨 번호 추첨 시 중복되지 않는 숫자 6개와 보너스 번호 1개를 뽑는다.
- 당첨은 1등부터 5등까지 있다. 당첨 기준과 금액은 아래와 같다.
  - 1등: 6개 번호 일치 / 2,000,000,000원
  - 2등: 5개 번호 + 보너스 번호 일치 / 30,000,000원
  - 3등: 5개 번호 일치 / 1,500,000원
  - 4등: 4개 번호 일치 / 50,000원
  - 5등: 3개 번호 일치 / 5,000원
- 로또 구입 금액을 입력하면 구입 금액에 해당하는 만큼 로또를 발행해야 한다.
- 로또 1장의 가격은 1,000원이다.
- 당첨 번호와 보너스 번호를 입력받는다.
- 사용자가 구매한 로또 번호와 당첨 번호를 비교하여 당첨 내역 및 수익률을 출력하고 로또 게임을 종료한다.
- 사용자가 잘못된 값을 입력할 경우 "[ERROR]"로 시작하는 메시지와 함께 `Error`를 발생시키고 해당 메시지를 출력한 다음 해당 지점부터 다시 입력을 받는다.

# 실행 흐름

0. 프로그램 시작
   - LottoController 인스턴스가 생성되며 프로그램 실행.
   - 내부에서는 LottoManager, InputProcessor 인스턴스 생성 인스턴스가 생성됨.
1. 로또 구입 금액을 입력 받는다

   - InputProcessor에서 금액을 입력받고, 입력값을 검증함.
   - 잘못된 금액을 입력할 경우, 에러 메시지가 출력되며 다시 입력값을 받음.

   **[검증]**

   - 입력값이 비어있는 경우.
   - 입력된 문자열이 숫자가 아닐 경우.
   - 1000원 단위로 입력하지 않은 경우.
   - 음수로 입력된 경우.
     → 오류 발생 시, 다시 금액을 입력.

2. 구입한 금액만큼 로또 발행

   - LottoManager가 받은 금액만큼 로또 수량을 계산 후 생성.
   - Lotto를 생성될 때, 로또 번호를 검증함.

   **[검증]**

   - 뽑힌 숫자가 6개가 인지.
   - 중복된 숫자가 없는지.
   - 뽑힌 숫자가 1~45 범위 내에 숫자인지.
     → 오류 발생 시, 로또를 다시 생성.

3. 발행한 로또 번호 출력
   - 발행한 수량만큼 로또 번호를 출력.
   - 오름차순으로 정렬하여 출력.
   - 로또를 "[1,2,3,4,5,6]"처럼 문자열로 출력.
4. 당첨 번호 입력

   - InputProcessor에서 당첨번호를 입력받고, 입력값을 검증함.
   - 잘못된 당첨번호를 입력할 경우, 에러 메시지가 출력되며 다시 입력값을 받음.

   **[검증]**

   - `,` 로 구분했을 때 6개의 숫자인지.
   - 모든 값이 정수인지.
   - 중복된 숫자가 없는지.
   - 모든 숫자가 1~45 범위 내에 숫자인지.
     → 오류 발생 시, 당첨번호를 다시 입력.

5. 보너스 번호 입력

   - InputProcessor에서 보너스번호를 입력받고, 입력값을 검증함.
   - 잘못된 보너스번호를 입력할 경우, 에러 메시지가 출력되며 다시 입력값을 받음.

   **[검증]**

   - 입력된 값이 숫자인지.
   - 모든 숫자가 1~45 범위 내에 숫자인지.
   - 당첨번호와 중복 됐는지.
     → 오류 발생 시, 당첨번호부터 다시 입력.

6. 당첨 결과를 계산

   - LottoManager가 로또 배열과 당첨번호를 받아 당첨 결과를 계산함.
     → 발행된 로또에서 몇개의 숫자가 (당첨 번호+보너스 번호)와 일치하는지 계산.
     → 일치된 개수에 따라 당첨 금액 및 총 수익률 계산.
   - 계산된 결과를 받아 LottoResult 인스턴스가 생성됨.

7. 당첨 통계 출력
   - LottoResult가 총 수익률 및 당첨 결과를 반환해줌.

- 계산한 당첨 통계와 총 수익률을 출력.
